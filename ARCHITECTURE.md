## 核心流程

### 1. 同步图片解析流程

```
用户上传图片
     │
     ▼
FastAPI 接收
     │
     ▼
验证 (类型/大小)
     │
     ▼
qwen-vl-max 识别 ──────▶ 重试 (最多 3 次)
     │                      │
     │                      ▼
     │                  降级 (qwen-vl-plus)
     ▼
返回 MedItem 列表
```

**耗时**: ~3-5 秒

---

### 2. Layer 1 安全性校验流程（新增）

```
成分列表 (来自图像识别) + 用户当前用药
     │
     ▼
RxNorm API 搜索
     │
     ▼
获取 RxCUI 标识符 ──────▶ 记录来源 URL
     │
     ▼
OpenFDA 查询 (并行)
     │
     ├──▶ 不良事件 (FAERS)
     │    └───▶ 记录来源：open.fda.gov/apis/drug/event/
     │
     ├──▶ 召回信息 (Enforcement)
     │    └───▶ 记录来源：open.fda.gov/apis/drug/enforcement/
     │
     └──▶ 标签警告 (SPL)
          └───▶ 记录来源：open.fda.gov/apis/drug/label/
     │
     ▼
DrugInteractionChecker
     │
     ├──▶ OpenFDA drug_interactions 文本查询
     ├──▶ LLM (qwen-max) 分析药物-补剂相互作用
     └──▶ 返回风险等级 (high/medium/low)
     │
     ▼
计算安全评分 (0-100)
     │
     ├── <50 ──▶ 拒绝，返回风险列表 + 来源 URL
     │
     └── ≥50 ──▶ 传递到 Layer 2/3
```

**耗时**: ~2-3 秒/成分

**来源 URL 示例**:
- RxNorm: `https://rxnav.nlm.nih.gov/REST/rxcui/{RxCUI}`
- OpenFDA 不良事件：`https://open.fda.gov/apis/drug/event/`
- OpenFDA 召回：`https://open.fda.gov/apis/drug/enforcement/`
- OpenFDA 标签：`https://open.fda.gov/apis/drug/label/`

---

### 3. 异步完整流程

```
用户上传图片 + 作息时间
     │
     ▼
FastAPI 接收并转 Base64
     │
     ▼
发送 Celery 任务 ──────▶ 立即返回 task_id
     │
     ▼
Redis 队列
     │
     ▼
Celery Worker 获取任务
     │
     ├──▶ 10% 开始解析图片
     │
     ├──▶ 40% 图片解析完成
     │
     ├──▶ 50% 安全性校验（RxNorm + OpenFDA）
     │
     ├──▶ 60% 开始生成排期
     │
     ├──▶ 70% 检索知识库 (RAG)
     │
     ├──▶ 80% 生成排期计划 (qwen-max)
     │
     ├──▶ 85% 检查药物-补剂相互作用
     │
     └──▶ 100% 完成
     │
     ▼
前端轮询 /api/task/{task_id}
     │
     ▼
显示结果
```

**耗时**: ~8-15 秒（用户可看到进度更新）

## 工程化特性

### 1. 速率限制 (Rate Limiting)
```
/api/parse-image          → 10 次/分钟
/api/generate-schedule    → 10 次/分钟
/api/init-knowledge-base  → 5 次/小时
```

### 2. 重试与降级 (Retry & Fallback)
```
主模型 (qwen-max) ──失败──▶ 重试 (指数退避) ──失败──▶ 降级 (qwen-plus)
```

### 3. 熔断器 (Circuit Breaker)
```
正常 (CLOSED) ──5 次失败──▶ 熔断 (OPEN) ──60 秒后──▶ 半开 (HALF_OPEN) ──成功──▶ 恢复
```

### 4. 输入验证
```
文件类型：.jpg, .jpeg, .png, .webp
文件大小：≤ 10MB
空文件检测：拒绝 0 字节
```

### 5. 异步任务队列
```
任务超时：300 秒
Worker 并发：4
重试次数：2 次
进度追踪：实时百分比
```

## 技术选型对比

| 模块 | 选型 | 备选 | 理由 |
|-----|------|------|------|
| 前端 | React + Vite | Vue/Svelte | Figma Make 导出，可高度定制 UI |
| 后端 | FastAPI | Flask | 异步支持，OpenAPI 文档 |
| 任务队列 | Celery | RQ | 功能丰富，支持进度追踪 |
| 消息代理 | Redis | RabbitMQ | 轻量，简单 |
| 图像识别 | qwen-vl-max | GPT-4V | 中文优化，成本低 |
| LLM | qwen-max | GPT-4 | 中文优化，成本低 |
| Embedding | text-embedding-v3 | ada-002 | 中文理解，成本低 |
| 向量库 | ChromaDB | Pinecone | 本地存储，零成本 |
